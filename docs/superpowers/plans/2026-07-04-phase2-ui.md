# Phase 2: UI + Data Visualization Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:executing-plans (inline) — implement sub-phase by sub-phase, full-green gate after each task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Rebuild the TokenTally UI as a five-mode shell (Chatbot/Prompt/Agent/Crew/Denial-of-Wallet) on the Phase-0/1 engine, with real-time tokenization in a web worker, honest cost + confidence + badge trust surfaces, seven data visualizations, a command palette, WCAG AA, and the §5.9 CSP/egress/E2E runtime verification actually running in-sandbox — deleting the old MVP.

**Architecture:** One Zustand store holds shell state + per-mode UI inputs + model-selection KEYS (not resolved records) + worker token counts + per-mode forecasts. The engine + registry are reached ONLY through two dynamic-import singletons, so the first-paint entry chunk stays engine-free (the `assert-first-paint-lean` gate greps it). Tokenization runs in one shared ES-module worker (worker-src `'self'`, no blob:). Charts are hybrid: hand-rolled SVG/DOM for waterfall/tornado/token-stream (exact control of the never-silent-`$0` branch), recharts-lazy (one shared chunk) for warmth-curve/scatter/step/blast-radius. Theme is CSS custom properties (script-src `'self'` forbids an inline theme script).

**Tech Stack:** React 18.3 + TS 5.6 strict, Zustand 4.5, Vite 7 (worker ES modules + manualChunks), Tailwind 3.4 (static CSS), recharts 2.12.7 (lazy), Vitest + Testing Library + Playwright (in-sandbox). No new runtime deps.

## Global Constraints

- **First-paint gate is sacred (P1-A25).** No first-paint module may top-level-import `@/workloads`, `@/optimization`, `@/engine`, `@/registry`, or `@/tokenizer`. The store imports only `zustand` + `import type` from `@/types/*`. Engine/registry/charts/worker/palette are all dynamic-import or `React.lazy` boundaries. `npm run first-paint-check` must stay green.
- **Never a silent `$0`** (C9/P1-A15). A `WarmCostResult` with `applicable:false`, a null cost component, an `unmodeled` confidence, or a non-`per_token` note renders its honest native-unit text, never a `$0`.
- **Honest copy (§6).** No unqualified "±5%", no "100% test coverage". The `honest-claims` gate scans the REAL shipped `dist` once the new UI ships — correct `index.html` + component strings.
- **CSP source of truth = the live `vercel.json`** (`style-src 'self' 'unsafe-inline'`, `script-src 'self'`, `worker-src 'self'`, no `unsafe-eval`). Design against it; `serve-with-csp.mjs` parses it so tests can't drift. Do NOT silently tighten style-src — escalate that as a separate hardening decision. Forbid `dangerouslySetInnerHTML`/`innerHTML`/injected `<style>` in the UI (eslint `react/no-danger`).
- **Real-time < 100 ms** via the worker + a 120 ms trailing debounce; forecast math is synchronous main-thread (sub-ms).
- **WCAG AA**: landmarks, roving-tabindex tablist, focus-trap + restore on modals, `aria-live=polite` (debounced) cost announcements, AA contrast in both themes, keyboard-reachable chart data tables.
- **TDD** (§8): Vitest unit tests for every store action, pure mapper, worker-client dedupe, and the never-`$0` branch; Playwright for CSP/egress/a11y/E2E. `npx tsc --noEmit` (base + scripts + tests) + `npm run lint` in every task's green step.
- **Pinned registry snapshot (RESOLVED blocker):** `scripts/registry/buildRegistry.ts` pins `PINNED_COMMIT = 59042a13a4932cdade3f3f352a81b27ec4b2557a` + `EXPECTED_SNAPSHOT_SHA256 = 15e09b288901e7a70909992d18aa82ba35b4485107c5a014a5c06b409e6f5359` (AgentOps-AI/tokencost `model_prices.json`, 2025-09-01, 1701 models, hash-verified). `SNAPSHOT_DATE=2025-09-01`. A `prebuild`/CI step runs `build-registry` before `tsc && vite build`. The generated `src/config/registry.generated.json` is git-ignored and CI-generated (or committed — decided in 2A Task 1).

## Cross-cutting decisions (from the design workflow, locked)

1. **State**: one `useAppStore` (Zustand); selection is a `(canonicalId, deployment)` key; static imports = `zustand` + `import type` only.
2. **Engine boundary**: `src/store/engineClient.ts` (`import('@/workloads')`/`import('@/optimization')`) + `src/registry/bootstrapRegistry.ts` (`import('@/config/registry.generated.json')` + `loadRegistry`) are the ONLY engine/registry entry points.
3. **Charts**: hand-rolled waterfall/tornado/token-stream; recharts-lazy for warmth/scatter/step/blast-radius; every recharts chart wrapped by `vizA11y` (role=img + aria-label + visually-hidden data table).
4. **Worker**: one shared ES-module worker (`worker:{format:'es'}`), same-origin file, tiktoken adapter only (Transformers.js is post-launch); `workerClient` monotonic id + pending map; `useTokenizer` 120 ms debounce + stale/model-mismatch drop.
5. **Lazy boundaries**: entry = shell + state + theme + landmarks; lazy = panels, engine, registry JSON, charts, worker, palette. `manualChunks`: react→`vendor`, recharts→`charts`.
6. **Theme**: CSS custom properties in the external stylesheet; `ThemeController` stamps `data-theme` in `main.tsx` before render; `prefers-color-scheme` covers the pre-JS frame.
7. **Tests**: Vitest (logic) + Playwright in-sandbox (CSP/egress/a11y/E2E, vendored same-origin axe).
